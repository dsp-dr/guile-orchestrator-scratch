* Overview

Guile Orchestrator is a distributed systems orchestration framework implemented in Guile Scheme 3.0+, originally targeting FreeBSD systems. This scratch/experimental project adapts orchestration patterns from modern distributed systems literature into a Scheme-based architecture using Guile's threading primitives, SRFI record types, and ~syntax-rules~ macros.

** Key Features

- Actor-based concurrency model using ~(ice-9 threads)~
- ~define-actor~ macro for pattern-matching message handlers
- Content-addressable storage with write-ahead logging
- OTP-style supervision trees (one-for-one, one-for-all, rest-for-one)
- Event sourcing with projections and snapshots
- Leader election, distributed locks, and vector clocks (in-process simulation)
- Circuit breaker and retry/timeout error handling
- Worker pool with stats and auto-scale monitoring
- Literate programming approach with org-mode

- FreeBSD 14.0+ or macOS (Linux untested but CI runs Ubuntu)

* Usage

** Quick Start

#+BEGIN_SRC scheme
(use-modules (orchestrator core)
             (orchestrator actors)
             (orchestrator tasks))

;; Define an actor
(define-actor worker
  ((process task)
   (format #t "Processing ~a~%" task)
   (task-complete! task)))

;; Create orchestrator instance
(define orch (make-orchestrator))

;; Submit a task
(submit-task orch 
  (make-task 'data-processing 
             #:payload '((input . "data.csv")
                        (output . "result.json"))))

;; Start orchestration
(orchestrator-run! orch)
#+END_SRC

** Declarative Orchestration

#+BEGIN_SRC scheme
(define-orchestration data-pipeline
  (stage extract
    #:actors 3
    #:handler extract-data)
  
  (stage transform
    #:actors 5
    #:depends-on (extract)
    #:handler transform-data)
  
  (stage load
    #:actors 2
    #:depends-on (transform)
    #:handler load-data))

(execute-orchestration data-pipeline input-data)
#+END_SRC

* Development Roadmap

** Pre-v1.0.0 Milestones

*** v0.1.0 - Core Actor System
- [X] Basic actor implementation (threads, not Fibers -- Fibers not yet integrated)
- [X] Message passing infrastructure (channel primitives)
- [X] Actor supervision tree (OTP-style supervisor)
- [X] Unit tests for actor system

*** v0.2.0 - Task Management
- [X] Task definition (record types, not DSL)
- [X] Task scheduling (basic, in core-enhanced)
- [ ] Priority queue implementation (stub only)
- [ ] Task persistence layer

*** v0.3.0 - State Machine Engine
- [X] FSM implementation (hash-table based, not delimited continuations)
- [X] State persistence and recovery (snapshot/restore)
- [X] Event sourcing integration (events module)
- [X] Snapshot mechanism

*** v0.4.0 - Distributed Coordination
- [ ] Gossip protocol implementation
- [X] Consensus algorithm (simplified Raft-like leader election)
- [X] Distributed lock service (with TTL)
- [X] Membership management (with heartbeats, vector clocks)

*** v0.5.0 - Storage Layer
- [X] Content-addressable storage (memory + disk backends)
- [X] Write-ahead logging
- [X] Compaction strategies (orphan removal)
- [X] Storage benchmarks (benchmarks/performance.scm)

*** v0.6.0 - FreeBSD Integration
